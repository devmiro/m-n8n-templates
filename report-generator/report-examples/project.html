Installation Guide

Prerequisites

Software Dependencies

  • Node.js version 18.0 or higher
  • PostgreSQL 14.0 or higher
  • Redis 6.2 or higher
  • Docker (optional but recommended)

Environment Setup

Development

Set NODE_ENV=development in your environment variables

Production

Configure SSL certificates and set NODE_ENV=production

Database

Create a new PostgreSQL database named 'taskflow_db'

Installation Steps

Clone the Repository

Command: git clone https://github.com/taskflow/taskflow-api.git

Download the latest version from our GitHub repository

Install Dependencies

Command: npm install --production

Install all required Node.js packages and dependencies

Configure Environment

Copy .env.example to .env and configure your database credentials

  • .env.example
  • .env.production

Database Migration

Command: npm run migrate:latest

Set up database tables and initial schema

Start the Application

Command: npm start

Launch the API server on default port 3000

API Endpoints

Authentication

Login

Method: POST

Endpoint: /api/v2/auth/login

Authenticate user and receive JWT token

Required Fields
  • email
  • password

Response Format: JSON with access_token and user_info

Register

Method: POST

Endpoint: /api/v2/auth/register

Create new user account

Required Fields
  • email
  • password
  • first_name
  • last_name
Optional Fields
  • company
  • phone

Project Management

Create Project

Method: POST

Endpoint: /api/v2/projects

Create a new project with initial settings

Required Fields
  • name
  • description
  • start_date
Optional Fields
  • end_date
  • budget
  • team_members

List Projects

Method: GET

Endpoint: /api/v2/projects

Retrieve all projects for authenticated user

Query Parameters
  • status
  • sort_by
  • limit
  • offset
Filters
  • active
  • completed
  • archived
  • on_hold

Update Project

Method: PUT

Endpoint: /api/v2/projects/{project_id}

Update existing project details

Updatable Fields
  • name
  • description
  • status
  • budget
  • end_date

Task Management

Create Task

Method: POST

Endpoint: /api/v2/projects/{project_id}/tasks

Add new task to specified project

Required Fields
  • title
  • description
  • priority
Priority Levels
  • low
  • medium
  • high
  • urgent

Assign Task

Method: PATCH

Endpoint: /api/v2/tasks/{task_id}/assign

Assign task to team member

Required Fields
  • assignee_id
  • due_date

Automatic email notification sent to assignee

Troubleshooting

Common Issues

Database Connection Failed

Symptoms
  • Connection timeout errors
  • Unable to connect to database
Solutions
  • Verify database credentials in .env file
  • Check if PostgreSQL service is running
  • Ensure database exists and user has proper permissions
  • Test connection using psql command line tool

Authentication Errors

Symptoms
  • 401 Unauthorized responses
  • Invalid token errors
Solutions
  • Check JWT secret key configuration
  • Verify token hasn't expired
  • Ensure proper Authorization header format
  • Clear browser cache and cookies

Performance Issues

Symptoms
  • Slow API responses
  • High memory usage
  • Timeout errors
Solutions
  • Enable database query optimization
  • Implement Redis caching for frequent queries
  • Monitor and optimize database indexes
  • Scale horizontally with load balancer
